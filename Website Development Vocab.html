The push method adds one or more elements to the end of an array and returns the length of the array.


let arr = [1, 2, 3];
arr.push(4);
console.log(arr); // [1, 2, 3, 4]

The pop method removes the last element from an array and returns it


let arr = [1, 2, 3];
let last = arr.pop();
console.log(last); // 3
console.log(arr); // [1, 2]

The shift method removes the first element from an array and returns that element. It also shifts everything else to al ower index


let arr = [1, 2, 3];
let first = arr.shift();
console.log(first); // 1
console.log(arr); // [2, 3]

Unshift adds one or more elements to the beginning of an array and returns the new length of the array. This also shifts all existing elements to a higher index.


let arr = [1, 2, 3];
arr.unshift(0);
console.log(arr); // [0, 1, 2, 3]

When using an array as a queue. One example could involve pushing an element to the back of an array and using shift to remove elements from the front.

let queue = [];
queue.push(1);
queue.push(2);
queue.push(3);
console.log(queue); // [1, 2, 3]
let first = queue.shift();
console.log(first); // 1
console.log(queue); // [2, 3]

When using an array as a stack, you can do the same but using push and pop.

let stack = [];
stack.push(1);
stack.push(2);
stack.push(3);
console.log(stack); // [1, 2, 3]
let last = stack.pop();
console.log(last); // 3
console.log(stack); // [1, 2]

JSON: JSON is lightweight and easy to parse. It is based on the JavaScript object syntax and it used on a lot of modern web applications for data exchange between client and server. JSON supports numerous data types (Strings, numbers, booleans, arrays, etc).

XML: XML is a markup language similar to HTML. It is flexible and extensible. Like JSOn it is also used in a lot of applications and industries like webdevelopment and document management. It supports different data types much like JSON.

YAML: YAML is human-readable. It is designed to be easier to read. Easier than XML or JSON. It is often used for configuration files and data exchange between different programming languages. It also supports numerous data types. But allows for nested objects and arrays.

Markdown: Markdown is lightweight and designed to be easy to read and write. It is often used to create formatted text for things like websites or blogs. It supports basic formatting, and advanced code blocks.

JavaScript is a scripting language that is commonly used to add interactivity to web pages. It is a high-level, dynamic, and interpreted language that runs in web browsers. TypeScript is a superset of JavaScript that adds static typing and other features to improve developer productivity and code maintainability. TypeScript code is transpiled into JavaScript code before execution.
Transpiling is the process of converting code from one language to another while preserving the functionality of the code. Useful for converting to a language that may add more functionality to a program.
ES6/7 refers to the sixth and seventh versions of the ECMAScript specification, which is the standard that JavaScript and TypeScript follow. ES6/7 introduced several new features to the language, including classes, arrow functions, and destructuring.
Node is a JavaScript runtime that allows developers to run JavaScript code outside of web browsers. It is built on the V8 JavaScript engine and provides APIs for working with files, network requests, and other common tasks. Node is not technically a framework, but it can be used with many different frameworks to build web applications. But it only stays within JavaScript's runtime environment. Think of it as a place to run JS code without putting it in HTML.

The 3 are all famous web development frameworks that let developers build web applications. But they contain a few differences.

Nextjs: It is a React-based framework. It profides features like code splitting, server-side rendering, static site generation, and automated optimized performence. It is best used for building data-driven web software that require a server-side for realtime updates.

Meteorjs: This framework provides most needed for building web software. Such as front-end framework, a back-end server, and a database. It lets developers write code in JavaScript for both client and server and makes it easier.

Expressjs: A minimalist framework for Nodejs. It provides simple and flexible tools for building web applications. The main focus is on provided basic web software infrastructure. It is suited best for single-page applications.

While they are the two most popular front-end JavaScript frameworks used for building websites, they share some differences.

For one, Angular is considered to be more difficult to learn than React due to syntax and architecture. React is lightweight and easier to learn with a simpler syntax.

When it comes to architecture, Angular is based on three layers. Model, controller, and view. All of these are included, while React only allows for the view layer. The rest must be added with other libraries. React's architecture is component-based. These components are rendered with React's DOM library.

Since react uses a virtual DOM. The performance is greatly improved. All DOM trees are lightweight and build on the server. Therefore, the load times are reduced. Angular performs a lot worse in comparison due to the complex and dynamic nature of Angular. Bindings in Angular are assigned watchers which check and track values. The more bindings you have, the more watchers, and with that comes a bigger beating on performance.

React requires numerous tools to test applications with it which makes testing each different component and JavaScript code a chore. Angular allows the use of only one tool which makes it sightly less straining to test.

Asynchronous function is a function that starts an operation and potentially returns before the operation completes. The operation completes in the background, allowing other code to execture in the meantime. Synchronous completes an operation before returning.
Blocking calls cause the program to wait until the requested operation is complete before continuing. Non-blocking calls allow the program to continue running while the requested operation is performed in the background.

  • HTTP if the protocol used for transferring data over the internet. 2 versions exist, HTTP/1.1 and HTTP/2.0 although 1.1 is the most common version
  • HTTP contains several methods used for making requests between the client and server.
    • GET requests data from a specified resource
    • POST submits an entity to be processed by a specified resource
    • PUT Uploads a representation of the specific resource
    • DELETE deletes a specified resource
    • HEAD Retrieves the headers for a specified resource
    • OPTIONS describes the communication options for the specified resource
    These are just a few methods, there are many more.
  • HTTP request and response messages are the communication messages that are exhanged between client and server. Request is sent from the client to request something from the server. Response is what the server sent back to the client
  • HTTP message header fields provide information about the request or response. Such as the length, type, software the requesti is made from, server the request is sent to, and the type of media.
  • Response codes are the three digit codes that indicate the status of the response send by the server to the cleint. Some common ones are:
    • 200: request was succesful
    • 404: request was not found
    • 500: error when processing request
    • 302: request was found but was moved
